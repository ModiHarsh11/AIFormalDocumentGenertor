API USAGE & INTEGRATION DETAILS
================================================================================

═══════════════════════════════════════════════════════════════════════════════

GOOGLE GEMINI API FLOW:

1. USER INITIATES GENERATION
   ├─ User enters:
   │  ├─ Document Type (office, circular, policy)
   │  ├─ Topic/Subject
   │  ├─ Language (en, hi)
   │  └─ Optional: Previous body (for regeneration)
   │
   └─ Request sent to Django view

2. DJANGO VIEW PROCESSING
   ├─ Validates input parameters
   ├─ Checks user authentication
   ├─ Loads appropriate prompt template from prompts/ folder
   └─ Calls service.generate_body() or service.regenerate_body()

3. PROMPT TEMPLATE SELECTION
   ├─ English prompt templates: prompts/office_order.py, circular.py, policy.py
   ├─ Hindi prompt templates: Same files (separate Hi_TEMPLATE_REGENERATE, etc.)
   ├─ Templates use LangChain PromptTemplate for variable substitution
   └─ Built in prompts/_shared.py

4. GOOGLE GEMINI API CALL
   ├─ Endpoint: https://generativelanguage.googleapis.com
   ├─ Authentication: GEMINI_API_KEY (from .env)
   ├─ Request Format:
   │  {
   │    "model": "gemini-2.5-flash-lite",  // or fallback model
   │    "contents": "Your prompt text here",
   │    "config": {
   │      "temperature": 0.3,
   │      "max_output_tokens": 1024
   │    }
   │  }
   │
   └─ Response: Generated text with AI-created document body

5. RESPONSE HANDLING
   ├─ Check if response is valid (not empty, not blocked by safety filters)
   ├─ Extract text from response
   ├─ Sanitization: Remove markdown formatting, validate content
   ├─ Validation: Check document quality and completeness
   └─ Return to user or raise error

6. FALLBACK MECHANISM
   ├─ If primary model (gemini-2.5-flash-lite) fails:
   │  ├─ Automatic retry with gemini-2.0-flash
   │  ├─ If that fails, try gemini-2.0-flash-lite
   │  └─ Continue until all models exhausted
   │
   └─ Error Types Triggering Fallback:
      ├─ Quota exceeded (too many requests)
      ├─ Model unavailable (503 error)
      ├─ Rate limit reached (429 error)
      └─ Connection timeout

7. OUTPUT DELIVERY
   ├─ Save to database (GeneratedDocument model)
   ├─ Display in UI
   ├─ Allow export to:
   │  ├─ .docx (Word document)
   │  ├─ .pdf (PDF file)
   │  └─ .html (HTML preview)
   └─ User can download or regenerate

API KEY SECURITY:
   ├─ Never hardcoded in source code
   ├─ Stored only in .env file (Git ignored)
   ├─ Read from environment at runtime via os.environ.get()
   ├─ Should be rotated regularly
   └─ Different keys for development vs production

RATE LIMITING:
   ├─ Google Gemini has rate limits based on API plan
   ├─ Application automatically handles 429 responses
   ├─ Fallback to alternative models if available
   └─ Consider implementing caching for frequently requested documents

COST CONSIDERATIONS:
   ├─ Each API call costs quota
   ├─ Text generation: ~ 1 credit per ~1000 input/output tokens
   ├─ Fallback provides reliability but uses more quota
   └─ Monitoring recommended to track usage


PROJECT STRUCTURE & DATA FLOW
================================================================================

═══════════════════════════════════════════════════════════════════════════════

USER AUTHENTICATION FLOW:
   1. User visits /login/ (auth_views.py)
   2. Enters username and password
   3. Django authenticates against User model
   4. Creates session (database-backed in production)
   5. Redirects to /dashboard/
   6. Session verified on each request via middleware


DOCUMENT GENERATION FLOW:
   1. User accesses form (e.g., /office-order/)
   2. Fills in topic, language, additional fields
   3. Submits form
   4. views/office_order.py handles POST request
   5. Calls common.generate_body_with_ai()
   6. Validates input via validation.py
   7. Loads prompt from prompts/office_order.py
   8. Sends to Google Gemini API via client.py
   9. API returns generated text
   10. Sanitizes output via sanitization.py
   11. Saves to GeneratedDocument model
   12. Renders template with result
   13. User can download as .docx or .pdf


DATABASE MODELS:

   User (Django built-in)
      ├─ id, username, email, password_hash
      └─ Used for authentication

   UserProfile (Custom extension)
      ├─ user (ForeignKey to User)
      ├─ role, department, designation
      └─ Additional user metadata

   DocumentLog (Legacy - audit trail)
      ├─ document_type (Office Order, Circular, Policy, Notice)
      ├─ language (en, hi)
      ├─ reference_id
      ├─ content (full document)
      └─ created_at (timestamp)

   GeneratedDocument (Primary - new system)
      ├─ document_type (office, circular, policy)
      ├─ user (ForeignKey to User)
      ├─ subject_en, subject_hi
      ├─ body_en, body_hi (AI-generated content)
      ├─ status (draft, approved, published)
      ├─ created_at, updated_at
      └─ All metadata for complete audit trail

================================================================================

   1. Python 3.9 or higher installed

1. NAVIGATE TO PROJECT DIRECTORY:
   cd D:\BISAG\DocumentGenerator-main\DocumentGenerator-main

2. ACTIVATE VIRTUAL ENVIRONMENT:
   Windows:  .\.venv\Scripts\activate
   Linux:    source .venv/bin/activate

TROUBLESHOOTING COMMON ISSUES
================================================================================

Issue 1: "ModuleNotFoundError: No module named 'weasyprint'"
   Solution: Install system dependencies
   Windows:  Install Visual C++ Build Tools
   Linux:    sudo apt-get install libcairo2-dev libpango-1.0-0 libpango-cairo-1.0-0
   Mac:      brew install cairo pango

Issue 2: "Could not connect to PostgreSQL database"
   Solution: 
      1. Check .env DATABASE_URL is correct
      2. Verify internet connection (cloud DB)
      3. Check firewall/proxy settings
      4. Verify credentials with neon admin console

Issue 3: "GEMINI_API_KEY not found"
   Solution:
      1. Check .env file has GEMINI_API_KEY = xxx
      2. Ensure .env is in correct directory
      3. Restart Django server after .env changes
      4. Check key hasn't expired in Google Console

Issue 4: "Static files not found" (production)
   Solution: python manage.py collectstatic --noinput

Issue 5: "Database tables don't exist"
   Solution: python manage.py migrate
